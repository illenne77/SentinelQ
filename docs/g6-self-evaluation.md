# G6 자기 평가 — 본인 사용 가치 연 50만원+ 검증

**게이트**: G6 (PREREG-0008 §5 / §7.1, ADR-0013 §Method G3)
**작성일**: 2026-05-16
**작성자**: 본인 (SentinelQ Phase 3 사용자)
**상태**: 초안 — G1·G2·G3 실증 완료 후 최종 판정

---

## 판정 기준

| 조건 | 기준 |
|---|---|
| **PASS** | 본인 사용 가치 합산 ≥ **500,000원/년** |
| **PASS → Phase 2 진행** | G1+G2+G3+G6 모두 통과 (PREREG §7.1) |

## 1. 시간 절약 가치

매년 5월 양도세 신고 시 수동으로 소요되는 시간과 자동화 후 절약되는 시간을 산정한다.

| 항목 | 수동 소요 시간 | 자동화 후 소요 시간 | 절약 시간 |
|---|---:|---:|---:|
| KIS 거래내역 조회·복사 | 30분 | 0분 (자동 fetch) | 30분 |
| 종목별 FIFO 손익 계산 | 120분 | 0분 (자동 계산) | 120분 |
| 국내·해외 손익통산 | 30분 | 0분 | 30분 |
| NTS 홈택스 입력 항목 정리 | 60분 | 5분 (CSV 출력 확인) | 55분 |
| 재확인·오류 수정 | 30분 | 5분 | 25분 |
| **합계** | **270분 (4.5시간)** | **10분** | **260분 (4.3시간)** |

> **[사용자 입력]** 위 시간 추정이 실제 본인 경험과 맞는지 확인 후 수정하세요.

### 시간 가치 환산

```
절약 시간: 4.3시간/년
시간당 기회비용: [사용자 입력] 원/시간
  참고 기준: 최저임금 약 10,000원/시간, 지식노동자 기준 30,000~100,000원/시간

보수적 기준 (30,000원/시간): 4.3 × 30,000 = 129,000원/년
중간 기준   (50,000원/시간): 4.3 × 50,000 = 215,000원/년
적극적 기준 (100,000원/시간): 4.3 × 100,000 = 430,000원/년
```

**소계 (보수적)**: 약 **129,000원/년**
**소계 (중간)**: 약 **215,000원/년**

## 2. 계산 정확도 향상 가치

수동 계산 시 발생 가능한 오류(과소 신고→가산세, 과다 신고→세금 손실)를 방지하는 가치.

| 오류 유형 | 발생 확률 (추정) | 금전 영향 | 기대 손실 |
|---|---:|---:|---:|
| FIFO 계산 실수 (양도차익 과소 신고) | 20% | 미납세액×10%+이자 | 가변 |
| FIFO 계산 실수 (양도차익 과다 신고) | 15% | 과납세액 | 가변 |
| 국내·해외 손익통산 누락 | 25% | 통산 놓친 손실×22% | 가변 |

> **[사용자 입력]** G2 실증 결과를 기입하세요.
> - CLI 산출 총 양도차익: ______ 원
> - 실제 NTS 신고 결과: ______ 원
> - 차이: ______ 원 (G2 판정: PASS / FAIL)
>
> G2 PASS (±100원 이내) → 계산 정확도 위험 실질적으로 제거됨.

**소계**: 오류 방지 효과 (G2 PASS 전제) — 정량화 어려움, 보수적 평가 배제.

## 3. 손익통산 최적화 가치

도구가 식별하는 손익통산 최적화(12월 손실 인식 권장)로 절약되는 세금.

| 항목 | 내용 |
|---|---|
| G4 시뮬레이션 | gain=10,000,000 / loss=-6,000,000 → 절세액 1,320,000원 |
| 본인 실제 시나리오 | [사용자 입력] 실제 보유 종목 미실현 손실 기준 계산 |

> **[사용자 입력]** 실제 보유 포트폴리오에 대해 `loss_harvesting.py`를 실행한 결과를 기입하세요.
> - 추천 손실 인식 시 절세 가능액: ______ 원/년 (12월 기준)
> - 도구 없이 인지 가능 여부: 예/아니오

12월 손실 인식 권장이 **1회만 실현**되어도 최소 수십만원 절세 가능 (gain>250만 전제).

**소계 (보수적)**: 사용 빈도 낮은 해를 가정해 **0원** (보수적 기준)

## 4. 세제 한도 추적 가치

IRP·연금저축·ISA 한도 추적으로 누락하는 세액공제를 방지하는 가치.

| 계좌 | 연 한도 | 세액공제율 | 최대 절세 |
|---|---:|---:|---:|
| 연금저축+IRP 합산 | 9,000,000원 | 16.5% | 1,485,000원 |
| ISA | 20,000,000원/년 | 비과세 500만 | 가변 |

> **[사용자 입력]** 현재 납입 현황을 기입하세요.
> - IRP·연금저축 연간 납입액: ______ 원
> - ISA 연간 납입액: ______ 원
> - 도구로 확인한 잔여 한도 활용 가능성: 예/아니오

이미 한도를 완전 채우고 있다면 이 항목의 추가 가치는 0.
한도를 모르고 미납입했다면 도구로 수십만원~수백만원 추가 세액공제 가능.

**소계**: 한도 현황에 따라 **0 ~ 1,485,000원/년** (가변)

## 7. ADR-0013 §Method 기대값 대비

ADR-0013 §Method `기대값 base case`는 본인 사용 가치를 연 55~370만원으로 추정했다.

| ADR-0013 추정 | 본 자기 평가 (보수적) | 비고 |
|---|---|---|
| 55~370만원/년 | 129,000원~ | ADR-0013은 손익통산·세제한도를 적극 활용하는 시나리오 포함 |

보수적 기준 129,000원은 ADR-0013 base case 하한(550,000원)의 약 23%.
→ G6 PASS를 위해서는 **손익통산·세제한도 항목 중 최소 1개**가 실현되어야 한다.
